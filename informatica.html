Materia: INFORMATICA

Modello Client Server nei siti

Quando navighiamo su un sito web, stiamo sfruttando un modello di comunicazione chiamato client-server. Esso invia una richiesta a un server, ovvero un computer remoto che contiene i dati e le risorse del sito.Il server riceve la richiesta, la elabora e restituisce al client una risposta, che può essere una pagina HTML, un file multimediale o dei dati. Questo scambio di informazioni avviene attraverso protocolli standard come HTTP o HTTPS (versione più sicura con cifratura)che regolano il formato, la struttura e l’ordine delle comunicazioni tra i due sistemi.

Comunicazione tra Frontend e Backend

Il frontend è la parte visibile del sito: l’interfaccia con cui interagiamo. È costruita con tecnologie come HTML, CSS e JavaScript.
Il backend, invece, lavora sul server e si occupa della logica, del salvataggio dei dati e delle risposte da inviare al client.
Le API REST permettono al frontend e ad altri sistemi di comunicare con il backend attraverso richieste come: GET, POST, PUT, DELETE. I dati scambiati vengono solitamente formattati in JSON, un formato leggero e facilmente leggibile. In alcuni casi si usa anche XML, sovrattipo dell’html ma è anche molto pesante.
